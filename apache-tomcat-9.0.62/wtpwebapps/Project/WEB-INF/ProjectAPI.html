Integration of CRITIC and ENTROPY in TOPSIS


This Project API implements luggage tracking using RFID tags by implementing multiple different RESTful services. Users will be able to access the homepage which lays out the multiple different functions of this web service. Users will be able to register and add their credentials so it is processed by the database. Upon validation, their data is added to the database. After a successful login, users will be able to select a flight and dropoff their luggage. This will generate an RFID tag upon dropoff which will add the users' luggage in the database. Afterwards, users may invoke the tracking service found on the homepage. By entering the corresponding RFID tag, details about the location of the luggage will be displayed to the user. Only authenticated users will be able to invoke the tracking, making each step of the process fundamental to the program.

Database Structure

This API processes data locally by interacting with a SQL Database in the Cloud. This data is then used to facilitate specific functions invoked by the web services. These include authenticating users so that they are successfully added to the database, dropping off their luggage, selecting flights stored in the database, and updating the location of their luggage via RFID tag. This is the database schema.


User.sql

UserID

 primary Integer
A unique identifier for each user.

Username

 required String
The username for a user when they login to the service.

Password

 required String
The password for a user when they login to the service.

Passenger.sql

UserID

 primary Integer
A unique identifier for each passenger.

FlightNum

 required String
A unique identifier representing the flight selected by a user on the flight selection web service.

Luggage.sql

UserID

 required Integer
The ID of the user that is associated with the luggage.

RFID

 primary String
The unique RFID tag of the luggage that is used for tracking via Luggage Tracking.

Location

 required String
The current location or address of the luggage obtained via RFID tag in Luggage Tracking.

Flights.sql

FlightNum

 primary String
The unique identifier of a flight that is used for Flight Selection.

DepartingAirport

 required String
The starting point a flight that is selected via Flight Selection.

ArrivingAirport

 required String
The current location or address of the luggage obtained via RFID tag in Luggage Tracking.

FlightStatus

 required String
The current status of a flight i.e on time, departed, etc. which can be viewed via Flight Selection.
