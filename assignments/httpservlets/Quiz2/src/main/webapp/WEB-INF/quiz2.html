Multi Criteria Decision Analysis


This MCDA (Multi Criteria Decision Analysis API) allows users to process input data in a JSON format to perform Topsis and Weighted Sum Model. Topsis and Weighted Sum Model fall under Multi Criteria Decision Analysis. These models seek to compute the optimal solution or alternative by selecting specific features that users may be interested in and applying a utility function that will evaluate those features. Among multiple different possibilities, Topsis and Weighted Sum Model can be applied to compute the best outcome. The data generated by the user will contain different alternatives and the best one can be selected using Topsis and Weighted Sum Model based on the data presented.

GET REQUEST

Retrieve output for both MCDA methods

This is a direct invocation of the web server assuming that the dataset has been uploaded successfully onto the web server via {POST}. Endpoint specification is provided depending on the type of output you want.

http://localhost:8080/Quiz2/?files={files}&options={options}


http://localhost:8080/Quiz2/Topsis?files={files}


http://localhost:8080/Quiz2/WSM?files={files}

Parameters

files

 required
The designated file to be processed by the MCDA algorithm(s) once it has been uploaded to the web server. Only the file name is required; the uploaded path is assumed by the web server. The default is "data.json" but file names will vary on input. Thus, it is required to ensure precision.

options

optional
The response format that dictates how much information is presented to the user. There are four different values, "basic", "topsis", "wsm", "both" which will present four different outputs depending on the option selected. "both" represents the standard output so if an improper or no option is selected, the standard output will be selected.

http://localhost:8080/Quiz2/?files=data.json&options=basic


http://localhost:8080/Quiz2/Topsis/?files=data.json


http://localhost:8080/Quiz2/WSM/?files=input.json

Fields in Input Data

Alternatives
The categorical decisions that are evaluated by the algorithm(s). The number of decisions must be the same as the number of entries.

Attributes
The number of features that each specific alternative have. Each attribute represents a value found in each entry. Each attribute must be a non-negative double value.

Weights
The weight is a value that represents the impact each attribute has in evaluating the score. The number of weights must be the same as the number of attributes. Each weight must be a non-negative double value where the sum cannot exceed 1.

Entries
An array values for each attribute. Each alternative must have one entry.

Type
A modifier in determining whether an attribute is evaluated by the algorithm(s). The type only has two values, "True" and "False". True indicates that this attribute is included and False means the attribute is excluded. The number of types must be equal to the number of attributes.

Classification
Exclusive to Topsis. This is a modifier that determines whether the ideal value for a specific attribute contains the highest or lowest value. Each classifier has two attributes, "Maximum" and "Minimum". Maximum indicates the ideal best is the minimum value and Minimum indicates the ideal best is the minimum value. The number of classifications must be equal to the number of types.
